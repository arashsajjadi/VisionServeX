# VisionServeX v3.21 — Sidecar & Blocker-Elimination Sprint (Final Report)

**Status: held branch. NOT tagged, NOT published to PyPI. Awaiting CI-green + owner approval.**

Mission: put maximum pressure on the remaining blocked/hidden models before any
public release — implement isolated sidecars where they unblock real models, push
segmentation and embedding fine-tuning honestly, and promote **nothing** without
live evidence. Everything still blocked is hidden with an *exact reproduced*
blocker.

## What was built

- **Generic sidecar architecture** (`src/visionservex/sidecars/{base,client,protocol,errors}.py`):
  HTTP client to isolated-env Docker sidecars, normalized request/response
  protocol, typed error hierarchy with `hf_…` token redaction, disabled-by-default.
- **Florence-2 sidecar** (`docker/florence2/`): FastAPI server; the `flash_attn`
  hard-import is dropped via a `get_imports` patch (Florence-2 falls back to eager
  attention on CPU).
- **OpenMMLab HTTP sidecar** (`docker/openmmlab/sidecar_app.py` + Dockerfile):
  `/predict/pose` now serves `rtmpose-m` via the `pose2d=human` auto-download
  alias; form-field + result-parser bugs fixed; FastAPI baked into the image.
- **Segmentation fine-tune** (`training/segmentation_finetune.py`):
  `finetune_sam_decoder` + `SamDecoderModel` — frozen-encoder SAM mask-decoder
  fine-tune; the decoder state-dict is the only trained artifact.
- **Deeper embedding fine-tune** (`training/embedding_finetune.py`): `head_type`
  = `linear | mlp`; reload reconstructs the right head architecture.
- **Readiness taxonomy** (`readiness/taxonomy.py`): 3 new honest states —
  `VLM_READY_LIVE_SIDECAR`, `INFERENCE_READY_LIVE_SIDECAR`,
  `SEGMENTATION_READY_LIVE_SIDECAR` — distinct from host-runnable `*_READY_LIVE`.
  A sidecar-live model supersedes its host technical blocker but **never** a
  legal / gated / weights block.
- **Capability fields** (`core/model.py`): `sidecar_supported/required/name/live/
  cpu_verified/gpu_verified`, `anastig_sidecar_visibility`, `fine_tune_kind`.
- **Anastig contract v3.21** (`docs/anastig_model_contract_v321.md` +
  `anastig_model_allowlist_v321.json`): disjoint primary partition of all 151
  models incl. the sidecar buckets, regenerated by `tools/qa/v321_build_artifacts.py`.

## Still blocked — exact reproduced blockers (kept hidden)

See `v321_resolution_matrix.json`. Reproduced 2026-06-18:

- **OneFormer dinat-large** → `DEPENDENCY_MISSING`: natten 0.21.6 is installed but
  `transformers.models.dinat` imports `natten2dav`/`natten2dqkrpb`, removed in
  natten ≥0.17. Needs a natten-≤0.15 + matching-torch sidecar. *Low value:
  `oneformer-swin-large` is already `SEGMENTATION_READY_LIVE`.*
- **OneFormer convnext-large** → `WEIGHTS_MISSING`: weights never released. Permanent.
- **RT-DETRv4** (4) → `CUSTOM_LOADER_REQUIRED`/`CATALOG_ONLY`: HF has only community
  manga fine-tunes; no official numbered release exists. Cannot operationalize
  weights that do not exist.
- **DEIMv2** (8) → `CUSTOM_LOADER_REQUIRED`: official checkpoints **do** exist
  (`Intellindust/DEIMv2_DINOv3_S_COCO`) — upgraded from "unverified" — but the
  DINOv3-backbone + DEIM-head is custom-architecture (raw state_dict needs the
  upstream repo loader). Remaining work: a DEIMv2 sidecar vendoring the repo code.
- **DEIM** (2) → `CUSTOM_LOADER_REQUIRED`: upstream training-repo architecture.

Gated: `sam3-base` stays `GATED_TOKEN_REQUIRED` (no token supplied; never tested,
never printed).
